# ADR-004: Frontend Framework

**Status:** Accepted

## Decision
SvelteKit (not Next.js) for the production frontend.

## Context
The user's specification listed both SvelteKit and Next.js. For a document-heavy, real-time verification UI with WebSocket streaming of per-claim results, SvelteKit's reactivity model and smaller bundle size are preferable.

## Consequences
- SvelteKit frontend will be generated.
- API contract remains identical if replacement with Next.js is required later.
