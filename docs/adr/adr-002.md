# ADR-002: Dual Verification Strategy (SQL + Vector)

**Status:** Accepted

## Decision
Route claims to either SQL verification or Pinecone semantic retrieval based on claim type.

## Context
Factual claims fall into two categories:
1. **Quantitative claims** ("revenue was $4.2B", "margin improved to 23.4%") → verifiable by exact SQL query against structured DB.
2. **Qualitative/relational claims** ("the company expanded into European markets") → no single SQL value; require semantic similarity against a corpus of verified facts.

## Implementation
spaCy NER + dependency parsing classifies each claim. Claims containing `MONEY`, `PERCENT`, `DATE`, `CARDINAL` entities with arithmetic relations are routed to the SQL verifier. All others route to Pinecone.

## Consequences
- Higher accuracy than using a single strategy.
- SQL results are deterministic and auditable.
- Vector results carry a similarity score that feeds into the final Confidence Score.
