# ADR-003: Cryptographic Confidence Score

**Status:** Accepted

## Decision
Confidence Score is a deterministic SHA-256 hash of the verification result set, not an LLM-generated number.

## Context
In regulated industries (finance, healthcare), a confidence score generated by a probabilistic model is itself untrustworthy. The score must be reproducible — given the same document and the same database state, the score must always be identical.

## Implementation
```python
input  = sorted list of (claim_id, status, db_value, similarity_score) tuples
hash   = SHA-256(JSON.dumps(input, sort_keys=True).encode())
score  = (verified_count / total_claims) * 100  [human-readable component]
anchor = f"{score:.1f}#{hash[:16]}"             [the displayed "Confidence Score"]
```

## Consequences
- Score is tamper-evident.
- Any alteration of a single claim result changes the hash.
- The hash can be stored alongside the report for future audit.
